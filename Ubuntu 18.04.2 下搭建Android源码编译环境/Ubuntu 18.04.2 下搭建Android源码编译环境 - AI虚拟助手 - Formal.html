    Ubuntu 18.04.2 下搭建Android源码编译环境

    转至元数据结尾
    • 由 Chaoao9 Li 李超创建, 最终由 王一鸣修改于大约7小时以前
    转至元数据起始



    1.准备阶段


      安装OpenJDK

        每个Android源码版本需要对应的OpenJDK，这里使用的是手机是MI Play，对应的MI UI源码版本名称是lotus，lotus对应Android版本为8.1，对应的OpenJDK版本为8。

        查找MI UI源代码版本名称的方法为：


          1.连接手上需要调试的小米手机至PC


          2.

    运行shell命令后将输出MI UI源码版本名称
    ?
    adb shell


         

          3. 搜索MI UI对应的版本信息：

          http://corgi.pt.miui.com/  



          左下，设备打包信息      


          4.安装对应的OpenJDK

    OpenJDK安装命令
    ?
    sudo add-apt-repository ppa:openjdk-r/ppa
    sudo apt-get update
    sudo apt-get install openjdk-8-jdk

     

      

      安装编译环境依赖包


    环境依赖包工具链安装
    ?
    sudo apt-get install libx11-dev:i386 libreadline6-dev:i386 libgl1-mesa-dev g++-multilib
    sudo apt-get install -y git flex bison gperf build-essential libncurses5-dev:i386
    sudo apt-get install tofrodos python-markdown libxml2-utils xsltproc zlib1g-dev:i386
    sudo apt-get install dpkg-dev libsdl1.2-dev
    sudo apt-get install libesd0-dev # 这个没装上，找不到包
    sudo apt-get install git-core gnupg flex bison gperf build-essential 
    sudo apt-get install zip curl zlib1g-dev gcc-multilib g++-multilib
    sudo apt-get install libc6-dev-i386
    sudo apt-get install lib32ncurses5-dev x11proto-core-dev libx11-dev
    sudo apt-get install libgl1-mesa-dev libxml2-utils xsltproc unzip m4
    sudo apt-get install lib32z-dev ccache


        设置工具包配置


    git 配置
    ?
    git config --global user.name "your name"
    git config --global user.email "your@example.com"


    切换JDK至OpenJDK
    ?
    sudo update-alternatives --config java
    sudo update-alternatives --config javac



        使用repo同步代码树

            1.

    安装
    ?
    #设置环境变量
    mkdir ~/bin
    PATH=~/bin:$PATH
    #下载repo工具并让它具有可执行权限
    curl https://storage.googleapis.com/git-repo-downloads/repo > ~/bin/repo
    chmod a+x ~/bin/repo
    #建立源码下载目录名字和位置随意
    mkdir  ~/WORK_DIRECTORY
    cd  ~/WORK_DIRECTORY



            2.去http://gerrit.pt.miui.com/ 添加ssh key以获得权限：

             

    生成ssh key
    ?
    ssh-keygen -t rsa -C "1240847112@example.com"

              命令执行完毕后会在"~/.ssh"目录下生成两个文件"id_rsa"、"id_rsa.pub"，分别是私钥和公钥，这里使用"id_rsa.pub"中的公钥。


            3.repo安装完毕后，需要关联源码工程链接，工程链接查看方式：

               google的版本

               MI UI的版本


            4.关联代码树链接并同步，MI UI内网环境下经测试大概需要2小时左右，google的版本视网速而定，源码大多在50g~100g之间

    MI UI lotus版本用例
    ?
    repo init -u ssh://v-lichao9@git.mioffice.cn:29418/platform/manifest.git -b dev -m v10-o-lotus-dev.xml --repo-url=ssh://v-lichao9@git.mioffice.cn:29418/tools/repo.git
    repo sync


            5.

    安装jack-server
    ?
    cd ~/WORK_DIRECTORY/prebuilts/sdk/tools
    ./jack-admin install-server jack-launcher.jar jack-server-4.11.ALPHA.jar
    ./jack-admin start-server

    2.生产阶段


       代码修改相关

          MI UI源码在线查看（内网环境）：http://opengrok.pt.xiaomi.com/

          google源码在线查看：http://androidxref.com/、https://www.androidos.net.cn/sourcecode

          导入AndroidStudio及Debug：https://www.cnblogs.com/Lefter/p/4176991.html


       编译相关


    设置编译配置
    ?
    #在源码根目录下，加载脚本配置
    . build/envsetup.sh
    #加载编译配置，加载后需要进行选择编译配置
    lunch


          编译配置后缀解释如下，这里我使用的是lotus+userdebuge的组合：      

            eng：工程机        

            user：最终用户机

            userdebug：调试测试机

      

          最后进行正式编译，由于Android源码体系庞大，编译整个系统需要耗费5~10小时，一般选择分模块编译（改了哪里编哪里）！

          在每一个模块的源码目录下，都有一个Android.mk文件，里面包含了模块代码的位置、模块名称、需要链接的动态库等一系列的定义。


    编译framework模块用例
    ?
    make framework

       

          以framework为例，在我的台式机上编译一次大概10分钟左右，根据Terminal生成的信息找到源码根目录out文件夹下的framework.jar。


    将修改好的文件推送至手机
    ?
    #获取root权限
    adb root
    #挂载设备
    adb remount
    #push编译好的文件，your_file_path是你编译好的文件，target_file_path是Android手机中原有的文件路径
    adb push <your_file_path> <target_file_path>
    #重启后生效
    adb reboot

          

          如果是纯代码的修改，一般会生成.jar包或者.jar.prof二进制文件 ，需要一起推送至原手机对应的位置，推送前最好使用pull命令将设备中的原文件进行备份或者使用AndroidStudio中的文件浏览器进行保存。如果是资    源文件的修改，一般会生成类似framework-res.apk，一起先备份再推送到设备里就好。


    3.坑儿

      Out of Memory error.

              

          在编译命令之前,修改prebuilts/sdk/tools/jack-admin文件,找到文件中的这一行:      

          JACK_SERVER_COMMAND="java -Djava.io.tmpdir=$TMPDIR $JACK_SERVER_VM_ARGUMENTS -cp $LAUNCHER_JAR $LAUNCHER_NAME"

          然后在该行添加-Xmx8g,如:

          JACK_SERVER_COMMAND="java -Djava.io.tmpdir=$TMPDIR $JACK_SERVER_VM_ARGUMENTS -Xmx8g -cp $LAUNCHER_JAR $LAUNCHER_NAME"

          这里网上大多的教程都推荐修改成4g，经过我的实际测试，4g之后还是会OOM，所以这里我的修改是8g。


           然后更新设置

    重启jack server
    ?
    cd ~/WORK_DIRECTORY/prebuilts/sdk/tools
    ./jack-admin kill-server
    ./jack-admin start-server



    ninja: build stopped: subcommand failed

    ninja failed with: exit status 1


    在当前用户的.bashrc文件中增加
    ?
    echo "export LC_ALL=C" >> ~/.bashrc
    加链
    ?
    sudo ln -s /usr/lib/i386-linux-gnu/mesa/libGL.so.1 /usr/lib/i386-linux-gnu/libGL.so

          

          所有错误修改完毕后，重启终端，重新执行编译envsetup.sh、lunch、make编译三指令进行编译
